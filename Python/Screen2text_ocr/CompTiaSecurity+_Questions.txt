解答例
・入力値のサニタイズ処理

解説

SQL injection は、Web アプリケーションの入力フォームなどを通じて悪意のある
SQL コードを注入し、データベースを不正に操作する攻撃手法である。 入力値
のサニタイズ処理は、特殊文字や SQOL コマンドをエスケーブまたは無効化する
ことで、SQL インジェクション攻撃を防ぐ基本的かつ効果的な対策となる。バ
ラメータ化クエリも有効な対策だが、既存のコードベースへの実装には大きな変
更が必要となる。 WAF は外部からの攻撃を検知・プロックできるが、アプリ
ケーションレベルでの根本的な脆弱性対策としては不十分である。ブラックリス
トによるフィルタリングは、新しい攻撃パターンへの対応が困難なため、完全な
防御策とはならない。

--- page 52 ---
解答例

・パラメータ化クエリの使用

解説

SQL インジェクション攻撃に対する最も効果的な対策は、パラメータ化クエリ
(プリペアドステートメント) の使用です。この手法では、SQL クエリのロジッ
クとデータを分離し、ユーザー入力を常にデータとして扱うため、悪意のある
SQL コードが実行されるのを防ぎます。 WAF も有効ですが、バイパスされる可
能性があり、HTTPS やクライアント証明書は通信の暗号化や認証に関する対策
であり、SQL インジェクションそのものの防止には直接的な効果がありません。

解答例

・DNS キャッシュボポイズニング攻撃を防止する DNSSEC

解説

この事例は典型的な DNS キャッシュポイズニング攻撃のシナリオである。 攻撃
者が DNS サーバーのキャッシュを改ざんし、正規の Web サイトへのアクセスを
攻撃者のサーバーに誘導することで、フィッシング詐欺などの攻撃を行う。
DNSSEC は DNS レコードにデジタル習名を付与することで、DNS レコードの改
ざんを防止し、オリジンの真正性を保証する。 このため、DNS キャッシュポイ
ズニング攻撃に対する最も効果的な対策となる。IPSec は通信の暗号化と完全性
を確保するプロトコル、S/MIME は電子メールの暗号化と習名に使用される規格、
HSTS は HTTPS 通信を強制するセキュリティポリシー
